FlashForth 5 Quick Reference for AVR Microcontrollers

Interpreter

The outer interpreter looks for words and numbers delimited by whitespace. Numbers are pushed onto the stack; words are looked up and acted upon. Names are limited to 15 characters. Some words are compile-time only.

Data and the stack

The data stack (S:) holds 16-bit cells. Functions get arguments from the stack and leave results there. A return stack (R:) is available for temporary storage.

Memory

Microcontroller has three memory contexts (Flash, EEPROM, SRAM) unified into a single 64kB address space.

AVR8 Memory map

$0000 – (RAMSIZE-1)SRAM
RAMSIZE – (RAMSIZE+EEPROMSIZE-1)EEPROM
($ffff-FLASHSIZE+1) – $ffffFlash

SRAM includes IO-space and SFRs. Flash high mark is set by boot area + FF kernel size.

Memory context

The P register

P can be used as variable or pointer, with for..next or otherwise.

!pStore addr to P. ( addr -- )
@pFetch P register. ( -- addr )
!p>rPush P to R, store new addr in P. ( addr -- ) R:( -- addr )
r>pPop R to P register. R:( addr -- )
p+Increment P by 1.
p2+Add 2 to P.
p++Add n to P. ( n -- )
p!Store x at P. ( x -- )
pc!Store c at P. ( c -- )
p@Fetch cell at P. ( -- x )
pc@Fetch char at P. ( -- c )

In a definition, always use !p>r and r>p for proper nesting.

Pictured numeric output

Formatted unsigned double-precision integer constructed at end of tib.

<#Begin conversion.

For example:

-1 34. <# # # # #s rot sign #> type

results in -034 ok.

Interaction with the operator

Interaction is via a serial port, typically UART1, at 38400 baud 8N1 with Xon/Xoff. The selected serial port is determined by vectors 'emit, 'key, 'key?.
